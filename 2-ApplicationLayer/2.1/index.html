    1.Principles of Network Application

    1. 网络应用的体系结构

    (1). 客户机服务器结构

     网络应用由成对的应用进程组成，这些进程之间通过网络互相发送报文。通常情况下，每对进程的一端标识为客户端，而另一端为服务器。

    • 客户端：间接性接入网络；不与其他客户机直接通信；由于不需要对外提供服务，可能使用动态 IP 地址。
    • 服务器：永久访问地址、域名；大量服务器、分布式、处理并发请求
     进程或应用程序与网络之间的 API 被称为套接字，应用可以通过套接字选择传输层协议和控制一部分参数。

    (2). 点对点结构

     主机之间是平等的，任意节点之间都可以直接通信，提供服务、接受服务；节点可以间歇性接入网络、改变 IP 地址。
     优点：高度可伸缩；缺点：难于管理。

    (3). 混合结构

     具体的例子为 P2P 文件共享平台 Napster：文件的传输使用 P2P，文件的搜索采用 C/S。

    2. 应用层需求

    (1). 可靠性

     某些应用可以容忍一定的数据丢失，如网络电话；
     某些应用需要100%可靠的传输，如文件传输、网络银行。

    (2). 时间、延迟

     某些应用需要提供时间保障，例如，发送方注入套接字的比特到达接收方不迟于 $100$ ms。这些应用在延迟足够低时才会有效，如网络游戏、因特网电话等。

    (3). 吞吐量、带宽

     有吞吐量要求的应用被称为带宽敏感应用，如大多数多媒体应用。而弹性应用 (elastic) 能够根据当时的带宽情况或多或少地利用可供使用的吞吐量，如 E-mail、文件传输、Web 传送。
