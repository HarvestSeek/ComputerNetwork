    4.Routing of Internet

    1. Internet 路由: 层次化路由

     将整个网络看成一张图过于理想化，实际使用中会遇到以下问题：

    • 规模：随着节点数增多，有关路由选择信息的通信、计算、存储的开销不可实现
    • 管理自治：每个 ISP 都需要能控制其网内的路由，如选择路由协议、控制流量如何被路由等

     因此，实际中，路由器通过自治系统 (Autonomous System, AS) 组织起来，每个系统由唯一的 AS 号标识。路由也因此变得层次化，分为 inter-AS 和 intra-AS，同一 AS 内运行相同的路由协议，也称内部网络协议 (interior gateway protocols, IGP)，不同系统内的协议可以不同； 为实现 AS 间的通信，每个 AS 都需要有一些网关路由器 (gateway router)。
     在 Internet 中， intra-AS 侧重性能，而 inter-AS 侧重策略。
     e.g. 假设 AS1 通过 AS 间路由协议学习到，子网 x 通过 AS2 和 AS3 均可达，那么路由器 1d 会根据热土豆路由策略，将分组发往最近的网关路由器。这个任务也是由 AS 间路由协议完成的。

    2. RIP 协议

     路由信息协议 (Routing Information Protocol, RIP) 是一种 AS 内部协议，使用距离向量路由算法：

    • 其距离度量为跳步数 (hops)，最大度量为 $15$ 跳步
    • 每隔 $30s$，邻居之间交换一次 DV（通告），每次最多通告 $25$ 个目的子网
    • 对任一节点，如果 $180s$ 内没有收到来自某个邻居的通告，就认为相应链路失效 ($\infty$)
    • 应用了毒性逆转技术

     由于 $16$ hops 就等同于 $\infty$，RIP 只适用于小规模网络。
     RIP 的路由表由名为 route-d 的应用层进程管理，通告报文周期性通过 UDP 数据报发送。

    3. OSPF 协议

     开放最短路径优先 (Open Shortest Path First) 协议是一种 AS 内部协议，使用链路状态路由算法：

    • OSPF 通告在整个 AS 范围内泛洪，其报文直接封装至 IP 数据报中
    • 每个路由器构造完整的 AS 拓扑图，利用 Dijskra 算法计算路由

     相比于 RIP 协议，OSPF 有如下优点：

    • 安全：所有 OSPF 报文可以被认证
    • 允许维护多条最短路径，有利于负载均衡
    • 对于不同链路，可以根据 IP 数据报中的 ToS 设置不同的度量，实现不同服务类型的数据在不同链路中分流
    • 由于持有全局信息，可集成单播路由与多播路由 (最小生成树)
    • 支持在单个 AS 内再分层为主干区 (backbone) 和局部区 (area)。area 之间的通信只知道到达对方的最短路径，而不知到对方区域的详细拓扑。区边界路由器 (Area Border Routers) 同时位于两个区域内，汇总到达所在区网络的距离，通告给其他 ABR。

    4. BGP 协议

     边界网关 (Border Gateway) 协议是最重要的 Internet 协议之一，它已成为标准的域间路由协议，将 Internet 中数以千计的 ISP 连接起来。
     BGP 协议主要实现以下两方面的功能：

    • 某个 AS 通告其他 AS 某个子网的存在，并附带路径信息（而不是距离）
    • 一个路由器可能知道到某个子网的多条路径，它通过 BGP 选择一条路由

    (1). BGP 路由通告

     考虑这样一个情境：路由器 3d 向右图中所有路由器发布子网 x 的可达性信息。在 BGP 中，每对路由器通过 $179$ 端口的半永久 TCP 连接交换报文。每个连接称为 BGP 连接/会话；跨越不同 AS 的连接称为外部 BGP (eBGP) 连接，而 AS 内部的连接称为内部 BGP (iBGP) 连接。


     通常，在连接不同 AS 的网关路由器的链路上，有一条 eBGP 连接，每个 AS 内部有多条 iBGP 连接，且 iBGP 连接不总是与物理链路相对应。

     在通告报文中，有两个重要属性：1. AS-PATH，包含通告经过的 AS 序列，表示可以通过 [ASa, ASb, ...] 到达某个子网；2. NEXT-HOP，指明了 AS-PATH 起始的路由器接口的 IP。BCP 路由器还利用 AS-PATH 防止通告环路：如果一台路由器在路径列表中发现自己的 AS，就拒绝该通告。
     以上面的情境为例，3a 首先向 2c 发送一个 eBGP 报文，其内容大致为 [AS3, x, 11.11.1.1]，随后 2c 向 AS2 中包括 2a 在内的所有其他路由器发送 iBGP 报文。当一个路由器获得新的前缀可达性时，就会更新它的转发表。2a 可以通过 eBGP 连接继续传播可达性信息，也可能不这样做。

    (2). BGP 路由选择

     当有多条路径可供选择，BGP 并未直接采用自私的热土豆路由，而是采用基于策略 (policy-based) 的路由：
     ① 传播过来的路径被制定一个本地偏好 (local preference) 值作为其属性，这个值完全取决于相应 AS 的网络管理员。BGP 将首先选择偏好值最高的路径。
     ② 如果 ① 相同，选择具有最短 AS-PATH 的路由 (仅仅是 AS 的跳数)。
     ③ 如果 ② 相同，那么 BGP 就根据 NEXT-HOP 和热土豆路由策略选择路径。
     ④ 如果仍剩余多条路由，根据 BGP 标识符选择。
     BGP 在热土豆路由之前，利用 AS-PATH 进行了过滤，一定程度上很可能会减小端到端时延。

     用上图所示的例子，说明 BGP 的基于策略的路由。A、B、C、W、X、Y 是 $6$ 个 AS，其中 A、B、C 是提供商网络，W、X、Y 是客户网络。
     ① 尽管 X 是一个双宿接入 ISP，它作为一个客户网络，不期待作为中间者转发由 B 到 C 之间的任何流量，因此，它不会向 B 通告任何到达货经过 C 的路径，例如 X-C-Y。
     ② 对于任一运营商网络来说，它在通告路径的时候，只会执行那些源或目的是自己的客户的通告，这是一条商业上的经验法则而非官方标准。例如，A 可能向 B 通告路径 A-W，B 也可能向 X 通告路径 B-A-W，但 B 绝不会向 C 通告路径 B-A-W。
