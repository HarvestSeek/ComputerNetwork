    1.Introduction to Transport Layer

    1. 传输层概述

    (1). 传输层功能

     传输层为运行在不同主机 (端系统)的进程提供了一种逻辑通信机制(端-端)，传输层协议只工作在端系统中。
     传输层协议的功能：
     ① 发送方：将应用层递交的消息分成一个或多个报文段 (segment)，并向下传给网络层
     ② 接收方：将接收到的报文段组装成消息，向上交给应用层。

    (2). 传输层和网络层

    • 网络层：提供主机之间的逻辑通信机制
    • 传输层：提供应用进程之间的逻辑通信机制，位于网络层之上，依赖于网络层服务

    (3). Internet 传输层协议

     TCP: 可靠按序的交付服务，流量控制、拥塞控制、面向连接。
     UDP: 不可靠的交付服务，“尽力而为”的模型，只实现传输层必要的服务，如多路分用、多路复用
     两种服务均没有延迟和带宽方面的保障。

    2. 多路复用和多路分用

    (1). 概述

     多路分用和多路复用是传输层必要的功能。如果某层的一个协议直接对应上层的多个协议/实体，则需要复用/分用。
     接收端进行多路分用：传输层依据头部信息将收到的 Segment 交给正确的 Socket，即不同的进程。
     发送端进行多路复用：从多个 Socket 接收数据，为每块数据封装上头部信息，生成 Segment，交给网络层。

    (2). 多路分用

     主机接收到 IP 数据报(datagram)，每个数据报携带源 IP地址、目的 IP 地址、一个传输层的 Segment；每个段携带源端口号和目的端口号。
     主机收到 Segment 之后，传输层协议提取 IP 地址和端口号信息，将 Segment 导向相应的 Socket。

    ① UDP: 无连接分用
     UDP 的 Socket 用二元组标识：(des-IP, des-Port)。如果两个 UDP 报文段有不同的 源 IP 或 源端口号，但目的 IP 和目的端口号都相同，那么它们通过相同的 Socket 导向相同的进程。
    ② TCP: 面向连接的分用

     TCP 的 Socket 用四元组标识：(des-IP, des-Port, src-IP, src-Port)，接收端用所有四个参数将 Segment 导向合适的 Socket。与 UDP 不同的是，两个具有不同 源 IP 或源端口号的到达报文段会被导向两个不同的 Socket。

    3. UDP: User Datagram Protocol

    (1). UDP 功能

     ① 在 Internet IP 协议基础上，增加了分用复用，并做了简单的错误校验。在传输层做错误检测符合“端-端原则”。
     ② 因为 IP 协议就是尽力而为 (Best effort) 模型，所以 UDP也是；它是不可靠的，可能会发生丢失、非按需到达。
     ③ UDP 是无连接的：发送方和接收方之间不需要握手，每个 UDP 段的处理之间没有关系、相互独立。
     ④ 用于容忍丢失、速率敏感的场合：流媒体应用、DNS、SNMP 等。

    (2). UDP 的优点

    • 无需建立连接，减少延迟，实现简单
    • 头部开销少
    • 没有拥塞控制，应用层可以增加可靠性机制、错误恢复机制

    (3). UDP 报文段的格式

    (4). UDP 校验和 (checksum)

     用于检测 UDP 段在传输中是否发生错误，如位翻转。

     发送方生成校验和：
     ① 将段的内容视为 16-bit 整数。
     ② 校验和计算：计算所有整数的和，进位加在整数的后面，最后将得到的值按位取反
     ③ 将校验和放入校验和字段

     接收方利用校验和：
      接收并比较；若不相等，则检测出错误；若相等，则未检测出错误 (不代表无错误)。
