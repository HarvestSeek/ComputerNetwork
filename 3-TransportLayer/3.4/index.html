    4.TCP：Flow and Congestion Control

    5. TCP: 流量控制

     当一个 TCP 连接收到正确的数据后，会将其放入接收缓存；上层应用并不一定会立即读取其中的数据，如果读取速率远小于发送方的发送速率，就会导致缓存溢出。TCP 为应用程序提供了流量控制服务，使得发送方的发送速率与应用的读取速率相匹配。
     TCP 发送方维护一个称为接收窗口的变量，记为 RcvWindow，表示接收方空闲空间的大小。接收缓存大小记为 RcvBuffer，并满足 LastByteRcvd - LastByteRead ≤ RcvBuffer。
     于是 RcvWindow = RcvBuffer - [LastByteRcvd - LastByteRead]。接收方会把当前的 RcvWindow 值放入接收窗口字段中并发给发送方，其初值设为 RcvBuffer。
     发送方维护两个变量 LastByteSent 和 LastByteAcked，并控制 LastByteSent - LastByteAcked ≤ RcvWindow，保证接收方不会缓存溢出。
     注意到，当接收方缓存已满，通知发送方 RcvWindow = 0，按照上面的逻辑，发送方会停止发送数据并阻塞。TCP 中规定，当接收方接收窗口为 0 时，发送方发送一个只有一个字节的报文段，这个报文段会被接收方确认，然后清空缓存、返回一个非 0 的 RcvWindow。
