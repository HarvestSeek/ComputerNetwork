    1.A Nuts-And-Bolts Description

    1. 基本概念

     计算机网络是通信技术和计算机技术紧密结合形成的产物。它是互连的 (connected)、自治的 (autonomous) 计算机的集合。
     Internet 是世界范围的计算机网络，它互连了数十亿台计算设备。端系统通过因特网服务提供商 (Internet Service Provider, ISP) 接入互联网。每个 ISP 都是独立管理的，较低层的 ISP 通过国际或国家的高层 ISP 互联，高层 ISP 由高速光纤链路互连的高速路由器组成。
     网络协议是为网络中进行数据交换而建立的规则、标准，它规定了通信实体之间所交换消息的格式、意义、顺序以及对于特定事件采取的动作。其三要素为：语法 (Syntax)、语义 (Semantics)、时序 (Timing)。因特网标准由因特网工程任务组 (Internet Engineering Task Force, IETF) 维护，IETF 的标准文档称为 RFC (Request For Comment)。
     计算机网络的结构组成包括：网络边缘 (network edge)、接入网络 (access networks)、网络核心 (network core)。

    2. 网络边缘

     互联网互连了数十亿台计算设备，包括传统的 PC、服务器以及非传统的手表、游戏机、电器、汽车等，它们都位于网络的边缘，一般统称为端系统或主机。
     主机又进一步划分为客户端和服务器，服务器通常是更强大的机器，用于存储发布 Web 页面、流视频、中继电子邮件等。

    3. 接入网络

     指将端系统物理连接到边缘路由器的网络。

    4. 网络核心

     网络核心的功能为数据交换。通过网络链路和交换机移动数据通常有两种方法：分组交换 (packet switching) 和电路交换 (circuit switching)。交换机包括路由器和链路层交换机。

    (1). 分组交换

     多数分组交换机在链路的输入端使用存储-转发传输机制 (store-and-forward)：交换机必须在接收到整个分组以后，才能开始转发这个分组。
     分组交换是指把报文拆分成数据包和头部信息的集合，交换过程需要对数据进行拆分和重组。它需要额外的开销：头部占用空间、拆分重组消耗的时间。
     分组交换使用统计多路复用：整个信道资源随机分配给某个发送方，占用时刻随机、占用时间长短取决于发送数据的多少。
     设报文共 $M$ bit，分组长度 $L$ bit，链路带宽 $R$ bps，跳步数为 $h$，路由器数为 $n$，则报文交付所需时间 $$T=\frac{M}{R}+(h-1)\cdot \frac{L}{R}=\frac{M}{R}+n\frac{L}{R}$$

    (2). 报文交换

     是分组交换的前身，其他机制与分组交换类似，转发时不进行拆分。

    (3). 电路交换

     电路交换的基本过程为 建立连接-通信-释放连接，每条链路通常包含多条电路。它最典型的特点是资源独占，连接占用的缓存和带宽资源不能被第三方使用，而分组交换是按需共享链路。
     链路中的电路通过多路复用实现。

    ① 时分多路复用 (TDM)
     将时间划分为一段段等长的时分复用帧，每个用户在每个帧中占用固定序号的时隙，每个用户的时隙周期性出现。

    ② 频分多路复用 (FDM)
     每个用户占用不同的带宽资源，用户在被分配到一个频带后，自始至终都占用这个频带。

    ③ 波分多路复用 (WDM)
     光的频分复用。

    ④ 码分多路复用 (CDM)
     既不划分时间，也不划分频率，每个用户被分配一个大小为 $m$ bit 的码片序列 (chipping sequence)，其中的 $0/1$ 用 $-1/+1$ 表示。各用户的码片序列是正交的： $$\frac{1}{m} \mathbf{S_i}\cdot \mathbf{S_j}= \begin{cases} 1& if\ i=j\\ 0& else \end{cases}$$  定义 编码信号 = 原始数据×码片序列：如果发送比特为 $1$，则直接发送码片序列；如果发送比特为 $0\ (-1)$，则发送码片序列的“反码”。
     假设链路支持 $i$ 条电路，原始数据序列为 $\lbrace d_1,\ ...,d_i\rbrace$，其中 $d_i$ 为第 $i$ 个用户要发送的比特，$\mathbf{S_i}$ 是他的码片序列。定义叠加向量 $\mathbf{P}=\sum_1^n d_i\cdot \mathbf{S_i}$。
     最后，用户通过计算 $\frac{1}{m} \mathbf{S_i}\cdot \mathbf{P}$ 解码。如果结果为 $1/-1$，说明发送的比特为 $1/0$；如果结果为 $0$，说明相应的用户没有发送数据。

    (4). 分组交换 vs. 电路交换

     分组交换的缺点在于可能产生拥塞，导致时延或丢包。它不适用于实时服务，因为其时延难以预测。它使得信道资源被充分共享，更适用于突发数据网络。
     电路交换适用于持续数据网络，在其他情况下它对资源的利用率低，比如某个用户以一定的概率处于活跃状态的情况。
