    6.Retrospective：A Web Page Request


     假设一名学生 Bob 将他的笔记本与学校的以太网交换机相连，并下载了一个 Web 页面: www.google.com。上图描述了这个场景，学校的路由器与 ISP comcast.net 连接，并且它为学校提供了 DNS 服务。假设学校的 DHCP 服务器运行在路由器中。

    1. Getting Started: DHCP, UDP, IP and Ethernet

    ① Bob 手动配置 IP 等信息，他的笔记本的 OS 生成一个 DHCP 请求报文，依次封装至 UDP 报文段、IP 数据报，且 (src, dest) = (0.0.0.0, 68; 255.255.255.255, 67)。

    ② 数据报封装至以太网帧，其 (src, dest) = (00-16-D3-23-68-8A; FF-FF-FF-FF-FF-FF)。随后该帧广播至局域网内，包括 DHCP 服务器。交换机借此自学习了一个表项。

    ③ 路由器在链路层接口收到该帧，首先抽取出 IP 数据报，目的地址 255.255.255.255 指示了这个数据报应当交由高层协议处理。随后，依次抽取出 UDP、DHCP 报文。

    ④ 路由器中的 DHCP 服务器根据请求报文的内容，判断 IP 等内容可用，回应一个 DHCP ACK 报文，报文中携带了这些配置信息。然后依次向下封装到链路层帧，其 (src, dest) = (00-22-6B-45-1F-1B; 00-16-D3-23-68-8A)。

    ⑤ 通过表项，交换机寻址到 Bob 的笔记本。

    ⑥ Bob 的笔记本抽取出 DHCP ACK 报文，提取出配置信息，并在自己的 IP 转发表中添加默认网关的地址。它将目的 IP 在 68.85.2.0/24 以外的所有数据报发往默认网关。

    2. DNS, ARP

     联网后，Bob 在浏览器中输入了 URL: www.google.com。

    ⑦ 为了发送 HTTP 请求，Bob 的浏览器需要生成一个 TCP 套接字；而这个套接字需要 www.google.com 的 IP 地址。

    ⑧ Bob 笔记本上的 OS 生成一个 DNS 查询报文，将字符串 www.google.com 放入报文的问题段中。然后依次封装进 UDP、IP 中。其 (src, dest) = (x, 68.85.2.101; 53, 68.87.71.226)。其中，DNS 服务器的地址是通过 DHCP ACK 报文获知的。

    ⑨ 继续向下封装成帧时，由于目的 IP 在子网外，需要网关路由器接口的 MAC 地址。因此，Bob 的笔记本构造一个目的 IP 为 68.85.2.1 的 ARP 查询报文，并广播至局域网内。

    ⑩ Bob 从 ARP 应答报文中抽取网关的 MAC 地址，成功构造出了包含 DNS 查询报文的以太网帧。

    3. Intra-Domain to the DNS Server

    ⑪ 网关路由器接收该帧，抽取出包含 DNS 查询报文的 IP 数据报，读取目的 IP 后，根据其转发表决定把数据报发往 Comcast 最左边的路由器。然后封装成帧，通过链路发送。

    ⑫ Comcast 接收到帧后，抽取 IP 数据报，根据目的 IP 和转发表确定出接口。它的转发表已根据域内路由协议和域间路由协议 (BGP) 填写。

    ⑬ 在包含 DNS 查询报文的 IP 数据报到达 DNS 服务器后，找到与 www.google.com 对应 IP(64.233.169.105) 的 DNS 源记录 (假设已缓存)。DNS 服务器构造回答报文，依次封装，然后反向转发至学校的路由器。

    ⑭ 最终，Bob 的笔记本从 DNS 报文中抽取出 www.google.com 的 IP 地址。

    4. Web C/S Interaction

    ⑮ Bob 的主机成功生成了 TCP 套接字，它首先要与谷歌服务器的 TCP 进行三次握手。因此，首先构造 TCP SYN 报文段，依次封装。最终，其 dest = (80, 64.233.169.105, 00-22-6B-45-1F-1B)。

    ⑯ 随后，通过域内和域间路由转发，谷歌服务器抽取出报文段，回应一个 TCP SYNACK 报文段，并反向转发。

    ⑰ 最终，Bob 的笔记本收到这个报文段后，就可以发送 HTTP GET 请求了。再经过类似过程的转发，当它从 HTTP 响应报文中抽取出 Web 的 html 网页时，浏览器就显示了网页。整个过程至此结束。
