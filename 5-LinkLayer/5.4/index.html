    4.Switched LANs

    1. 链路层寻址：ARP 协议

    (1). MAC 地址

     主机和路由器的接口 (适配器) 除了具有一个 IP 地址以外，还具有一个链路层地址，称为物理地址、LAN 地址或 MAC 地址。MAC 地址通常为 $6$ 个字节，表示为“-”分割的六个十六进制数。
     MAC 地址是唯一的，在网卡出厂时就写入到了它的 ROM 中。当一个网卡商要生产 NIC 时，它向 IEEE 支付费用，购买一块 $24$ 位的地址空间。一般来说，对任意设备，无论到哪里使用，其 MAC 地址是不会改变的。

    为什么不能用 IP 代替 MAC

     其本质在于模块化、层次化的优点。两种地址面向不同层次的功能，如果耦合在一起，将带来很多不便。
     首先，链路层局域网必须可以支持多种网络层协议，如 IPX、DECnet 等，而不仅仅是 IP。
     其次，如果 NIC 使用网络层地址，它难以满足网络层功能需要的动态改变 IP 的需求，并且，NIC 需要处理多种网络层协议的报头，增加了 NIC 实现的复杂度。
     因此，对于不同层次的功能，各自使用自己的寻址方案是更好的设计。

    (2). ARP 协议

    ① 子网内通信

     如图所示，假设主机 C 要向主机 A 发送 IP 数据报，但主机 C 只知道目的 IP 而不知道目的 MAC 地址。于是，主机 C 查询自己的 ARP 表，ARP 表的格式如下：

    IP 地址MAC 地址TTL
    222.222.222.22188-B2-2F-54-1A-0F13:45:00
    222.222.222.2235C-66-AB-90-75-B113:52:00

    表中包含了 IP 到 MAC 的映射关系，TTL 记录了表项被设置的时间，通常其寿命大约为 $20$ 分钟。
     如果 C 查询到目的 MAC，则这个问题就被解决了；如果 ARP 表中没有相应表项，发送方就会构造一个 ARP 分组 (ARP packet)，目的 MAC 被设置为广播地址 FF-FF-FF-FF-FF-FF，局域网的每个节点都会检查自己的 IP 是否与 ARP 请求分组的目的 IP 是否匹配，如果匹配，则返回相应的映射。
     ARP 协议是即插即用的：ARP 自动建立而不需要手动配置，如果某主机与局域网断开连接，它的相应表项也会被清空。
     ARP 协议是一个介于网络层与链路层之间的协议。

    ② 子网间通信

     如图，假设主机 $1$: 111.111.111.111 向主机 $2$: 222.222.222.222 发送一个数据报。主机 $1$ 首先检测出目的 IP 不与自己在同一子网内，通过 ARP 获得网关的 MAC 地址后，向网关发送此数据报；之后，路由器查询转发表发送该分组，并将其封装到一个新的帧中，此时，帧中的目的 MAC 是已通过 ARP 获得的主机 $2$ 的 MAC。

    2. 以太网

     以太网 (Ethernet) 自 20 世纪 70 年代被发明以来，不断演化与发展，直至现在仍保持支配地位。它是应用最广泛的有线 LAN 技术：造价低廉，一个以太网 NIC 不足 ￥100；它比令牌局域网和 ATM 更简单、便宜；它可以满足网络带宽的需求。它以古时的假想物质“以太”命名。
     20 世纪 90 年代以前，以太网使用总线拓扑结构，同过同轴电缆总线互连节点，是一种广播局域网；90 年代后期，出现了基于集线器 (hub) 的星形拓扑，但其本质也是广播局域网；21 世纪初，位于中心的集线器被交换机所替代。

    (1). 以太网服务

     以太网提供无连接、不可靠的服务，遇到差错帧直接丢弃，丢弃帧的数据恢复依赖于高层协议。
     以太网的 MAC 协议采用 CSMA/CD。当监听的链路处于占用状态，以太网的推迟策略采用 $1$-坚持 CSMA；当在传输数据过程中没有检测到冲突，则确认发送帧成功发送；否则，发送方发送一个阻塞信号后中止发送，然后根据二进制指数退避算法确认等待的时间量：

    • 设这个帧经历了 $m$ 次的连续冲突，取 $n=\min{(m,10)}$
    • NIC 从 $\lbrace0,1,...,2^{n-1}\rbrace$ 中随机选取一个数 $K$，等待 $K\cdot 512$ bit 的传输延迟时间后，重新进入 监听-空闲时传输 的过程
    其原因在于，冲突次数多意味着网络负载高，应当提升随机时间的范围，减小节点之间连续冲突的概率。

    (2). 以太网帧的结构

      

    • 前导码 (Preamble)(8B): 前 $7$ 个字节均为 10101010，用于同步接收端和发送端的时钟；第 $8$ 个字节为 10101011，用于定界。
      • 时钟同步：同步串行通信需要通过相同的时钟频率恢复数据信息。虽然双方可以根据链路类型指定频率，但实际传输时会有微小偏差，因此需要同步。
      • 若不做特殊声明，前导码长度不算进帧长内。
    • 目的 MAC、源 MAC (各 6B): 接收方会匹配目的 MAC 和自身的 MAC，如果一致才会接收，否则丢弃。
    • 类型 (Type)(2B): 用于复用和分用，指示了上层协议，如 ARP 为 0x0806。
    • 数据 (min:46B, max:1500B): 最小值基于 CSMA/CD 的限制，当小于 $46$ 字节时需要填充，网络层通过 IP 的长度字段去除填充；最大值取自以太网的 MTU。
    • CRC (4B): 循环冗余校验码。
     以太网的最小帧长为 $\mathrm{64(+8)}$ 字节。
     以太网标准的不同通常是速率和物理介质的不同，帧的格式和使用的 MAC 协议是相同的。

    (3). 交换机

     交换机的功能是转发链路层帧，它对于子网中的主机和路由器来说是透明的 (transparent)。在交换以太网中，主机和路由器通过独享链路直连交换机，每段链路通过 CSMA/CD 收发帧，且不会有冲突出现；每段链路都支持全双工通信，链路之间支持并行通信。类似于路由器，交换机的输出接口也设有缓存。

     交换机的功能通过存储于内部的转发表 (switch table) 实现，其形式如下：

    MAC 地址接口时间
    01-12-23-34-45-56213:45
    .........

     基于交换表，交换机实现它的以下功能。
     ① 自学习：假设交换表为空，每当有一个帧到达交换机，交换机会记录源 MAC、帧到达的接口、到达时间为一个新的表项。如果某个表项超过了交换机设定的老化期，则删除相应表项。因此，交换机是即插即用的，不需要管理员的外部干预。
     ② 过滤和转发：假设某个帧到达交换机的 $x$ 接口，可能出现三种情况：

    • 表中没有对应表项，则交换机向除 $x$ 以外的所有接口广播该帧；
    • 交换机中有对应表项，但与接口 $x$ 关联，则交换机认为源与目的在同一个局域网网段，无需转发，将帧丢弃；
    • 交换机中有相应表项，且与接口 $y\ne x$ 关联，则交换机将帧放至 $y$ 的输出缓存。

    (4). 虚拟局域网

          

     如图是一个机构如中小企业、学校等的典型局域网的简化拓扑，部门之间通过交换机的互连而互连，整个机构通过路由器划分为一个 IP 子网。子网内任意一个接口如果发送一个广播帧 (如 ARP、DHCP)，这些交换机会把广播帧转发到所有接口上，因此整个子网是一个广播域。
     这样的配置常常会有以下缺点：

    • 现实中可能出于性能或者部门之间的安全隐私角度，需要划分多个广播域；
    • 如果每个部门的主机都较少，无法充分利用交换机的接口；
    • 属于多个部门的主机、部门之间主机的移动等问题解决起来困难，灵活性低。
     为实现广播域的划分，可以引入路由器划分子网，但这不是最经济的解决办法。
     虚拟局域网 (VLAN) 技术因此而产生。管理员可以通过软件对支持 VLAN 的交换机的接口进行划分，每一组为同一个 VLAN，每个 VLAN 的端口形成一个广播域。基于端口的划分是在软件层面通过交换表实现的，类似地还以基于 MAC 地址划分。
     如果要实现 VLAN 之间的通信，通常需要在不同 VLAN 的端口间再连接一个路由器，对于集成了路由器功能的三层交换机实现这一点更为方便。

     如果 VLAN 的划分需要通过多交换机实现，就需要连接这些交换机。一种连线的方法是把跨越不同交换机的 VLAN 中的某两个端口连线，如上图所示。其问题在于不具有扩展性，对于 $2$ 台交换机和 $N$ 个 VLAN，将需要连接 $N$ 条线。

     更好的方法是连接交换机之间的中继端口 (trunk port)，经过 trunk 端口转发的帧必须附加 VLAN 的 ID，IEEE 专门为此设置了 802.1q 协议。
     这里只是对 VLAN 作简要介绍，实际的 VLAN 也可以基于网络层而定义、跨越 IP 路由器扩展。
